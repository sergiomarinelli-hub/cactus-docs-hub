Produto

Visão geral

O Queue Hub centraliza filas de trabalho no Jira. Times criam visões por JQL, organizam filas em grupos prioritários e salvam preferências individuais de uso.

Filas por JQL Cada fila usa uma consulta JQL configurável e limitada ao projeto atual.
Colunas configuráveis Usuários podem ajustar colunas, densidade, ordenação e preferências de visualização.
Governança Admins controlam acesso, acompanham logs de uso e exportam dados operacionais.

Produto

Funcionalidades principais

Criação de filas por JQL Admins definem consultas JQL para montar filas por projeto, time ou cenário.
Colunas e densidade personalizadas Cada pessoa ajusta colunas, largura, densidade e idioma sem impactar o time.
Grupos prioritários Filas podem ser organizadas no menu lateral por grupos, deixando o trabalho mais escaneável.
Templates de filas Modelos prontos aceleram a criação de filas recorrentes, como My work, Unassigned e Waiting customer.
Abertura rápida de issues Abra chamados pela chave da issue ou com duplo clique na linha.
Exportações Filas e logs podem ser exportados em CSV para análise externa, auditoria ou acompanhamento operacional.

Manual

Como usar o Queue Hub

O app tem duas áreas principais: visão operacional de filas e administração. Usuários consomem filas e ajustam preferências; admins configuram filas, grupos, permissões e auditoria.

  1. Abrir o app no projeto Jira Acesse a aba Queue Hub dentro do projeto Jira.
  2. Escolher uma fila Use o menu lateral para navegar entre grupos e filas. O contador mostra a quantidade atual.
  3. Criar ou editar fila Admins definem nome, JQL, grupos e colunas.
  4. Personalizar visualização Defina idioma, densidade, favorita e largura de colunas.
  5. Exportar dados Use Exportar CSV na fila ou nos logs para análise externa.
Fila operacional Exibe issues do JQL com colunas configuradas e abertura direta.
Preferências pessoais Salva idioma, densidade, favorita, sidebar, colunas e seleção atual.
Templates Cria filas a partir de modelos recorrentes como My work e Unassigned.

FAQ

Perguntas frequentes

Quem pode criar ou editar filas?

A criação e edição ficam disponíveis para usuários com permissão de gerenciamento configurada nos perfis de acesso do app.

Como uma fila define quais chamados aparecem?

Cada fila usa um JQL próprio. O app executa essa consulta no contexto do projeto Jira e retorna as issues visíveis para o usuário.

As colunas são iguais para todo mundo?

A fila tem colunas padrão, mas cada usuário pode salvar preferências pessoais de largura e visualização sem alterar a experiência do time.

O idioma é por usuário?

Sim. A preferência de idioma é salva por usuário e alterna a interface entre PT-BR e EN-US.

Para que servem os logs administrativos?

Os logs ajudam org admins a acompanhar abertura do app, visualização de filas, abertura de issues e erros recentes.

Quando usar templates de filas?

Templates aceleram a criação de filas comuns, como trabalho do usuário atual, itens sem responsável e filas baseadas em espera ou SLA.

Produto

Visão geral

O Issue Action Hub concentra ações recorrentes da issue em um app Atlassian Forge. Ele aparece no painel da issue e como ação modal, permitindo alterar status por transições do workflow, editar campos disponíveis e publicar comentário externo no Jira Service Management.

Ações no contexto da issue O app identifica a issue atual pelo contexto do Forge e recarrega dados após cada alteração.
Execução operacional Transições e campos são enviados ao Jira com validação, tratamento de erro e feedback ao usuário.
Rastro de auditoria Registra eventos em Forge KVS e pode criar comentários automáticos de auditoria na issue.

Produto

Funcionalidades principais

Painel e ação modal Disponível como jira:issuePanel e jira:issueAction, com modal grande para fluxos de ação.
Alteração de status Lista as transições disponíveis no workflow e executa a transição selecionada pela API do Jira.
Campos obrigatórios da transição Quando o workflow exige campos adicionais, o app exibe o formulário antes de executar novamente.
Edição de campos Usa o editmeta do Jira para montar editores de texto, número, data, seleção, usuário, lista e JSON.
Responsável operacional Quando o assignee é recusado pelo Jira, registra o responsável operacional e tenta atribuir a issue ao usuário técnico do app.
Comentário externo Publica resposta pública no portal JSM e usa comentário público da issue como fallback em cenários permitidos.

Manual

Como usar o Issue Action Hub

Use o app quando uma issue precisar de uma ação rápida e rastreável: mudar status, preencher campos, responder o cliente ou combinar mais de uma dessas alterações em uma única rodada operacional.

  1. Abrir a issue no Jira Acesse o painel Issue Action Hub ou a ação Issue Action Hub no menu de ações da issue.
  2. Atualizar os dados Use Atualizar para recarregar status, transições, campos editáveis e permissões retornadas pelo Jira.
  3. Alterar status Escolha a próxima transição. Se houver campos obrigatórios, preencha o formulário exibido.
  4. Editar um campo Selecione um campo editável, informe o valor e salve. Campos não suportados pelo Jira ficam ocultos.
  5. Publicar comentário externo Escreva a mensagem para o cliente e publique como comentário público no portal do Jira Service Management.
  6. Salvar ações combinadas Quando houver mais de uma ação pendente, use Salvar todas as alterações para executar o conjunto.
Alterar status Seleciona transição, valida campos obrigatórios e atualiza a issue no workflow.
Editar campo Atualiza campos editáveis com conversão para o payload esperado pelo Jira.
Comentário externo Envia resposta pública e mantém o autor operacional no corpo da mensagem.

FAQ

Perguntas frequentes

Onde o app aparece no Jira?

Ele aparece como painel da issue e também como ação de issue em modal grande.

Quais ações o app executa?

O app altera status por transições do workflow, edita campos liberados pelo Jira e publica comentário externo no JSM.

O que acontece quando a transição exige campos?

O app exibe os campos obrigatórios conhecidos. Se o Jira apontar campos adicionais no erro, o formulário é enriquecido e a pessoa tenta novamente.

Quais tipos de campo são suportados?

Texto, textarea, ADF, número, data, datetime, select, multiselect, lista de strings, usuário e JSON para campos complexos.

Como funciona o responsável operacional?

Se o Jira recusar a atualização do assignee, o app registra o responsável operacional no campo configurado ou no storage interno e tenta usar o usuário técnico como assignee.

Como o comentário externo é publicado?

Primeiro pela API do Jira Service Management. Em erros 400, 403 ou 404, o app tenta criar um comentário público na issue.

Que auditoria é registrada?

Transições, alterações de campo e comentários externos recebem registro no Forge KVS; transições e campos também podem gerar comentário automático na issue.
