Produto

Visão geral

O Queue Hub centraliza filas de trabalho no Jira. Times criam visões por JQL, organizam filas em grupos prioritários e salvam preferências individuais de uso.

Filas por JQL Cada fila usa uma consulta JQL configurável e limitada ao projeto atual.
Colunas configuráveis Usuários podem ajustar colunas, densidade, ordenação e preferências de visualização.
Governança Admins controlam acesso, acompanham logs de uso e exportam dados operacionais.

Produto

Funcionalidades principais

Criação de filas por JQL Admins definem consultas JQL para montar filas por projeto, time ou cenário.
Colunas e densidade personalizadas Cada pessoa ajusta colunas, largura, densidade e idioma sem impactar o time.
Grupos prioritários Filas podem ser organizadas no menu lateral por grupos, deixando o trabalho mais escaneável.
Templates de filas Modelos prontos aceleram a criação de filas recorrentes, como My work, Unassigned e Waiting customer.
Abertura rápida de issues Abra chamados pela chave da issue ou com duplo clique na linha.
Exportações Filas e logs podem ser exportados em CSV para análise externa, auditoria ou acompanhamento operacional.

Manual

Como usar o Queue Hub

O app tem duas áreas principais: visão operacional de filas e administração. Usuários consomem filas e ajustam preferências; admins configuram filas, grupos, permissões e auditoria.

  1. Abrir o app no projeto Jira Acesse a aba Queue Hub dentro do projeto Jira.
  2. Escolher uma fila Use o menu lateral para navegar entre grupos e filas. O contador mostra a quantidade atual.
  3. Criar ou editar fila Admins definem nome, JQL, grupos e colunas.
  4. Personalizar visualização Defina idioma, densidade, favorita e largura de colunas.
  5. Exportar dados Use Exportar CSV na fila ou nos logs para análise externa.
Fila operacional Exibe issues do JQL com colunas configuradas e abertura direta.
Preferências pessoais Salva idioma, densidade, favorita, sidebar, colunas e seleção atual.
Templates Cria filas a partir de modelos recorrentes como My work e Unassigned.

Engenharia

Documentação técnica

O Queue Hub é um app Atlassian Forge com Custom UI em React. A interface roda no iframe do Forge, chama resolvers via @forge/bridge e usa Jira API/Forge KVS para dados e configurações.

Runtime

  • Atlassian Forge
  • Custom UI com Vite
  • Resolvers Forge em TypeScript
  • Storage via Forge KVS

Frontend

  • React 18
  • TypeScript
  • CSS próprio do app
  • Atlaskit icons, lozenge e tokens

Qualidade

  • Vitest para testes unitários
  • tsc --noEmit para typecheck
  • vite build para build da UI
  • forge lint antes de deploy

Fluxo de dados

React UI @forge/bridge invoke Forge resolver Jira REST API / Forge KVS Resposta normalizada para a tabela

Código

Mapa do código

src/app Shell da aplicação, rotas internas, i18n, estado global e preferências do usuário.
src/components Componentes React: sidebar, toolbar, tabela, editor de filas, logs admin e telas de configuração.
src/features/queues Tipos, API client, seeds, normalização de colunas, JQL e regras de armazenamento de filas.
src/features/issues Adaptação de issues Jira para linhas de tabela, formatadores e exportação CSV.
src/forge Resolvers Forge, integração com Jira REST API, storage KVS e permissões.
src/shared Contratos compartilhados entre frontend e resolver, especialmente payloads de invoke.

FAQ

Perguntas frequentes

O login protege as docs?

Em GitHub Pages o login é apenas uma barreira de interface. Para segurança real, publique em repositório privado ou use uma camada com autenticação corporativa.

Como exportar PDF?

Clique em Exportar PDF e use a opção de salvar como PDF na janela de impressão do navegador.

Como adicionar novos apps?

O layout já está preparado para múltiplos apps. Basta adicionar uma nova entrada no menu e novas seções de documentação.
